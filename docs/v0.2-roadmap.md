# cc-bridge v0.2 — Contracted Handoff 路线图

**状态:设计定稿(2026-06-22)。**
验证来源:对抗性多 agent sweep(21 条净新增漏洞)+ Codex 跨模型复审(确认 C1、修正 C5/#11、新增 #22)。

## 目标

把「一个 agent 给另一个 agent 发一段自由文本任务」升级成「提交一份**可验证、权限不可扩张**的委派合同」。当前 `codex_execute` / `claude_analyze` 以自由文本为输入,`ContextBuilder` 把项目树/README/manifest 拼进 prompt,最后要求对方自然语言总结——这是全项目最薄弱的接口。

## 非目标(本版明确不做)

GUI 工作台、容器平台、自动学习路由、余额预测、用户历史推荐、复杂任务阶段分类器、多层 agent 群聊。**签名 token 也不做**(见 F3):一个被明确定义为「非安全边界」的机制只会制造虚假安全感,且 `executor` 当前 `os.environ.copy()` 会把任何 env 密钥直接泄给子进程。

## 安全核心:三脚架(缺一塌)

1. **本地重授权** — `effective_scope = requested ∩ 父继承scope ∩ local_user_policy ∩ engine_limits`,每次 handoff 无条件;`requested` 只是申请、永不授权;`engine_limits` 只当上限。
2. **跨进程事务边界** — 崩溃可释放的 OS 句柄锁 + 外部 WAL 日志,跨 `snapshot → exec → verify → rollback` 持有。
3. **可验证证据** — 进程树静默后哈希磁盘原始字节、越界检测、逐文件回滚、按类别副作用向量。

### 不变量(任何 PR 不得违反)

- `requested ≠ allowed`;缺省 = **deny**。
- token / 链路自述 ≠ 授权;唯一真边界是 `local_user_policy ∩ engine_limits`。
- containment 是**检测 + 补救**,不是阻断沙箱;回滚是**补偿**,不是隔离。
- `verified` 只覆盖「静默后、外部存证、原始字节」可证的内容;其余标 `unknown` / `taint`。
- 只有**一条**强制路径到 `run_codex` / `run_claude`;不得有第二条绕过 policy/evidence 的路。
- 不诚实标注 `enforcement`(网络/命令等无法真强制时返回 `unsupported`,不假装)。

## Definition of Done

- `requested` 不能通过 handoff 扩大父任务权限。
- README/manifest 内容不能修改有效目录、网络策略或审批策略。
- 输入与输出都有版本化 schema(MCP Inspector 可见)。
- 写出作用域外文件 → 结果明确失败 + 列违规文件 + taint。
- 失败转交另一 agent 前,必须证明 `side_effect_state == none`。
- 旧 `codex_execute` / `claude_analyze` 仍可用,但不绕过 policy/evidence。
- 同一任务用「自由文本」vs「结构化」各跑 **N 次**(LLM 随机),比越界率与 check 通过率,结构化明显更稳。
